# Карта типов ОГЭ: examType → uiKind

Справочник для разработки и наполнения контента.

- **examType** — номер задания в экзамене (1–23), кодификатор ФИПИ.
- **uiKind** — шаблон отображения и проверки на сайте (их **9**, не 23).

Полная карта типов, шаблонов UI и правил автопроверки.

**Проверка карты (фаза 0 миграции):** `npm run validate:oge-phase0` → [`phase-0-report.md`](phase-0-report.md). Подробнее — [`MIGRATION.md`](MIGRATION.md).

## Правила автопроверки подробнее

### `twoChoice` (типы 1, 5, 6, 8, 11, 13, 14)

- Выбрать **ровно 2** утверждения из списка (5 или 6 пунктов — параметр задания).
- Порядок номеров не важен.
- Засчитываются галочки, ячейки или оба способа при согласованном наборе цифр.

### `matchTriple` (типы 4, 9, 10, 12, 15, 17)

- Три буквы (А, Б, В) сопоставляются цифрам из правой колонки.
- Число вариантов справа (2–5) — параметр задания, шаблон один.

### `orderedDigits` (типы 3, 7)

- Ученик записывает цифры в ячейки **в заданном порядке** (порядок элементов или «сначала X, затем Y»).

### `multiChoiceFour` (тип 16)

- Всегда 4 суждения с чекбоксами и **4** ячейки ответа.
- В ключе может быть **1, 2, 3 или 4** верных номера — длина массива `correct` определяет это для конкретного задания.
- Ограничение «не больше N галочек» **не** ставить.

Пример JSON:

```json
"answer": { "correct": ["2", "3", "4"] }
```

### `numericInt` и `numericMassTable` (типы 18, 19)

- Эталон хранится как число уже в нужной точности (`16`, `1.6`, `8.48` и т. д.).
- Проверка: `parseFloat(ввод) === value` — **без** `Math.round`, `Math.abs` и допусков.
- Замена «.» на «,» в поле — через `OGE_answerDotsToCommasInField`, как сейчас.

### `openReference`, `experimentOpen` (типы 20–23)

- Кнопки «Проверить» нет; разбор в блоке `solution`.

## Связанные задания 18 и 19

В одном варианте ОГЭ задания 18 и 19 часто используют общий контекст (одно вещество). При хранении в JSON можно связать их полем `meta.linkedTaskId`.
