# Рекомендации для агентов и контрибьюторов (chemistry-school)

Ответы пользователю — на **русском**. Код и коммиты — понятные, без лишнего шума.

## Нумерация заданий ОГЭ

- **Карта типов → шаблоны UI и правила проверки:** [`data/oge/TASK-TYPES.md`](data/oge/TASK-TYPES.md) (9 шаблонов отображения для 23 типов экзамена). Автопроверка: `npm run validate:oge-phase0`.
- **JSON-задания (фаза 1 миграции):** `data/oge/tasks/{id}.json`. **Варианты:** `data/oge/variants/{slug}.json` → `pages/oge/variants/`. Генерация: `npm run generate:oge-variants`, сборка: `npm run build:oge`.
- **Тип задания** (кодификатор ФИПИ): число **1…23** — в тексте «Задание 5», «Задание 12» и т. д.
- **Глобальный номер примера** — сквозной по всем когда-либо добавленным примерам на сайте. Хранится в `data/oge-registry.json` (поле `id`), не обязан совпадать с номером типа.
- На **странице типа** заголовок каждого примера: **`Задание {тип} № {глобальный id}`** (например «Задание 1 № 1001»). По клику открывается **отдельная страница только этого примера**: `pages/oge/ex/{id}.html`.
- Исходные шаблоны одного примера на тип (`oge-01.html` … `oge-23.html`): базовый набор (в реестре без `sourceDir`) — в **`data/oge-source/default/`**; остальные наборы — в **`data/oge-source/2026-demo/`**, **`variant-1/`**, **`variant-2/`**, **`variant-3/`**, **`variant-4/`**, **`variant-5/`**, **`variant-6/`** и т. п. **Сборка страниц:** `npm run build:oge` (из JSON в `data/oge/tasks/`). Legacy из HTML: `npm run build:oge:legacy`.
- Новый пример: добавить запись в `data/oge-registry.json`, при необходимости расширить скрипт сборки или завести шаблон в подпапке **`data/oge-source/`**, затем снова **`npm run build:oge`**.

## Структура каталогов

- **`pages/oge/`** — список типов `index.html`, страницы типов `type-01.html`…`type-23.html`, отдельные примеры в **`pages/oge/ex/{id}.html`**.
- **`pages/topics/`** — темы и `index.html` (список тем).
- **`pages/tests/`** — страницы тестов.
- **`pages/tables.html`** — справочные таблицы.
- **`index.html`** (корень) — входная точка сайта.
- **`css/`**, **`js/`**, **`assets/`** — общие ресурсы; относительные пути смотреть по глубине вложенности (`../../` из `pages/oge/ex/`).

## HTML

- Семантика: `header`, `main`, `article`/`section`, корректная иерархия заголовков.
- Классы для ссылок на сквозной номер: **`oge-task-seq`** (стили в `css/style.css`).
- Интерактив: подписи, `aria-label` на кнопках и полях где уместно.

## JavaScript

- В общем коде сайта и в **`js/`** используйте **`const`** и **`let`**, не **`var`**.
- Инлайн-скрипты в **`data/oge-source/**/oge-\*.html`** после правок тоже на **`const`/`let`** (сборка ожидает `const correct = [...]`для страниц`oge/ex`).
- Современный DOM API; избегать устаревших паттернов без причины.
- Форматирование: **Prettier с настройками по умолчанию** (`npm run format`). Не «сжимать» файлы всухую — сохранять читаемость как у Prettier.

## Добавление контента ОГЭ

- Учитывать, что **шаблоны различаются** по типам; при необходимости объединять похожие (например типы 9 и 10).
- Дублирование разметки допустимо, если проще поддерживать; альтернатива — вынести генерацию в `scripts/` и хранить данные отдельно.

### Задания на соответствие (колонки «А/Б/В — 1,2,3…»), типы **4, 9, 10, 12, 15, 17**

- У обоих списков в блоке `.match-columns` на `<ul>` задайте класс **`oge-match-plain`**, чтобы у позиций не было маркеров-«точечек» (см. `css/style.css`).
- Пример: `<ul class="oge-match-plain"><li>А) …</li>…</ul>`.

### Два верных ответа из списка с чекбоксами и ячейками (**как задание 1**), типы **1, 5, 6, 8, 11, 13**

- В скрипте объявите **`const correct = ["m", "n"];`** — два номера **по возрастанию** (строками), как в ключе ФИПИ.
- Число пунктов списка передайте последним аргументом: обычно **5**.
- Проверку делайте через **`OGE_twoChoiceAllOk(chosen, c1, c2, correct, optionCount)`** из `js/oge-check-feedback.js`: засчитываются только галочки, только две ячейки, или оба способа, если наборы цифр совпали и галочки верны.
- Ограничение «не больше двух галочек» — как сейчас через `limit()` на чекбоксах в этих шаблонах.

### Задание **16** (1, 2, 3 или 4 верных суждения из 4)

- У каждого суждения должен быть чекбокс.
- Под заданием всегда ставьте **4** ячейки ответа.
- Не задавайте ограничение числа выбранных чекбоксов: лимит подсказывает, сколько верных ответов в конкретном задании.
- В скрипте объявите **`const correct = [...]`** по возрастанию и проверяйте через **`OGE_multiChoiceAllOk(chosen, cells, correct, 4)`** / `OGE_eachMultiChoiceScope(...)`.

### Кнопка «Ответ» на `pages/oge/ex/`

- Сборка подставляет готовый ответ по полю **`const correct`** или по условиям в скрипте; при добавлении новых задач с двумя номерами сохраняйте **`const correct = [...]`** для предсказуемой строки «ответ …».

### Задание **19** (масса элемента по массовой доле)

- У **каждого** примера этого типа — и на **`pages/oge/type-19.html`**, и на отдельной странице **`pages/oge/ex/{id}.html`** — обязательна **таблица для самостоятельного хода решения**: блок **`div.oge-mass-table-wrap`** с **`table.oge-mass-table`** (строки «элемент» / «вещество», колонка «W, %», редактируемые ячейки), как в существующих шаблонах **`data/oge-source/**/oge-19.html`**. При оцифровке новых заданий типа 19 не оставляйте только поле числового ответа без этой таблицы; после правок шаблона выполните **`npm run build:oge`\*\*.
